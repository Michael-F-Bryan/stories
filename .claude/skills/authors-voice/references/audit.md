# Fresh-context prose audit

Audit the final prose only after developmental and canon grounding. Open this file in a fresh context.

This job is diagnosis, not a rewrite pass. Find material reader-visible defects, not opportunities to maximize craft.

When you report a finding, include:

- exact text and location
- registry rule ID and provenance
- reader consequence
- severity and materiality
- smallest repair direction
- owning layer or escalation

Use explicit `good as-is` when there is no material issue.

Separate observation from repair. Preserve slack and unaffected prose. Avoid rewriting the whole chapter.

Re-audit only materially changed passages after repair.

If the issue is structural, route it to story-engine. If it is canon, knowledge, or spoiler handling, route it to serial-process.

When you need a rule definition or exception scope, consult `references/rule-registry.md`.
