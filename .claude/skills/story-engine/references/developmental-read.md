# Developmental read

A developmental read is a structural diagnosis, not a prose edit and not a canon adjudication.
It looks only for causal movement, agency/choice, character-specific pressure, promise/payoff movement, reader anticipation, and structural repetition.

## Required finding format
For each finding, include:

- exact quotation
- reader consequence
- materiality
- owning layer
- `good as-is` when no change is needed

## Owning layer
Name the layer that should handle the fix:

- `story-engine` for shape, pressure, and promise movement
- `authors-voice` for line-level prose
- `serial-process` for ledger/state/spoiler issues

Do not use this read to rewrite text.
Do not use this read to check canon.
Do not use this read to propose line edits.

## Diagnostic focus

### Causal movement
Does the beat cause a real change, or does it merely restate what we already know?

### Agency / choice
Does the character make a meaningful choice under pressure, or are they only carried along?

### Character-specific pressure
Is the pressure specific to this person, or could any character sit in the beat with the same result?

### Promise/payoff movement
Does the scene advance, complicate, or pay off an existing promise in a way the reader can feel?

### Reader anticipation
Does the beat increase pull, redirect desire, or sharpen the next question?

### Structural repetition
Is the scene repeating the same dramatic shape, same exit note, or same kind of turn too often?

## Reporting rule
If the answer is no, say so plainly and attach the reader consequence.
If the answer is yes and the beat is already working, mark it `good as-is`.
Keep the read short, exact, and actionable.
